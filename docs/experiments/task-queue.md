# Task Queue

Deferred items from autonomous post-train-eval audits + multi-lane combo brainstorms. When a node is free and the autonomous loop has no urgent work (no fresh lane finished, no eval running short), pull items from this queue.

## Queue format

Each entry:

- `[<lane_tag>] [<date>] [<source>] [<finding>] [<proposed fix>] [<trigger condition>] [<estimated effort>]`

`<source>` ∈ {audit, combo, user-request}. `<trigger condition>` describes when this item becomes actionable (e.g., "after free node + 4h slot", "after 081 verdict published").

## Priority ranking

P0: blocks frontier work (rare; mostly audit fundamental fails — those halt loop, not queue)
P1: meaningful EV, queue when free
P2: nice-to-have, queue with low priority
